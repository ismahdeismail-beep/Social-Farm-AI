# DASHBOARDS

## Purpose
Dashboards provide role-based, actionable views of platform performance, enabling data-informed decision-making.

## Dashboard Profiles
| Profile | Focus | Key Views |
| :--- | :--- | :--- |
| **Executive** | Business Growth | ARR, Workspace Growth, Platform-wide ROI |
| **Marketing** | Campaign Performance | Engagement Rate, Conversion Rate |
| **Content** | Content Performance | Reach, Retention, Virality |
| **Operations** | System Health | Uptime, Error Rates, Pipeline Latency |

## Design Principles
- **Action-Oriented:** Dashboards must clearly indicate *what* action to take (e.g., "Performance low - review campaign").
- **Concise:** Avoid clutter; highlight the most critical KPIs.
- **Interactive:** Allow filtering by date, platform, and workspace.
- **Consistent:** Use the [VISUALIZATION_GUIDELINES](./VISUALIZATION_GUIDELINES.md) for charts and colors.
