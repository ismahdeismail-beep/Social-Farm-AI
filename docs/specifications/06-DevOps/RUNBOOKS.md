# Runbooks

## Scenarios
*   **Application won't start**: Check logs, resource limits, configuration.
*   **Database unavailable**: Check connectivity, service status, logs.
*   **Redis unavailable**: Check connectivity, service status.
*   **AI provider failure**: Implement fallback, check API status.
*   **Storage unavailable**: Check connectivity, permissions.
*   **Publishing queue stuck**: Check worker status, queue depth.
*   **High CPU/Memory**: Analyze logs, scale resources.
*   **Disk full**: Clean up logs, expand storage.
*   **Failed deployment**: Rollback, check logs.
